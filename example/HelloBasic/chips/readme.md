# Chips 示例

## 0. 为什么设计这个控件

`chips` 面向“快速筛选”场景，用户需要在有限空间内高频切换单个激活标签（例如 全部/打开/关闭/异常）。  
单独抽象为控件后，业务层可以直接使用清晰语义接口（如 `set_selected_index`、`clear_selection`、选择回调），避免每个页面重复写选择状态管理逻辑。

## 0.1 为什么不用现有控件

- `button_matrix`：提供的是通用矩阵点击能力，缺少筛选标签语义和默认选择约束。  
- `toggle_button`：只适合单个二值开关，不适合一组互斥筛选项。  
- `tab_bar`：偏页面导航语义，作为内容区轻量筛选器会偏重。

设计取舍：渲染层复用 `button_matrix` 以降低实现成本，同时通过 `chips` 暴露独立接口和功能边界。

## 1. 概述

本示例用于验证 `chips` 在标签筛选场景下的可用性与可视一致性。

## 2. 视觉与布局规范

- 控件尺寸：`220x96`  
- 网格：3 列、2 行，共 6 个 chips  
- 间距：6px

## 3. 控件清单

| 变量名 | 类型 | 尺寸 | 初始状态 | 目的 |
|---|---|---|---|---|
| `chips_view` | `egui_view_chips_t` | 220x96 | 预置 6 个标签 | 快速筛选 |

## 4. 状态覆盖矩阵

| 状态 | 覆盖说明 |
|---|---|
| normal | 默认渲染 |
| pressed | 按下/抬起反馈 |
| selected callback | 点击后回调 selected index |
| boundary | 最后一个 chip 点击路径 |

## 5. 录制动作

录制顺序点击 chips 索引：`1 -> 4 -> 5`。

## 6. 验收标准

- Runtime check 通过。  
- 交互转换帧可见。  
- baseline 晋升后，回归对比通过。

## 7. 已知限制与下一步

- 当前实现为单选点击流程。  
- 多选与持久化策略计划在后续迭代补齐。  
- 未来可扩展可选模式配置（single/multi）。

## 8. 功能重叠与差异化边界

- 与 `button_matrix` 在底层绘制和点击处理上存在重叠。  
- 差异化目标是“筛选标签”语义、接口命名与默认行为约束，避免把通用矩阵逻辑暴露给业务层。

## 9. 回归录制配置说明

- 推荐录制参数：`clock-scale=6`、`snapshot-stable-cycles=1`。  
- 本示例要求选中态对比度在所有录制帧中保持可辨识。
