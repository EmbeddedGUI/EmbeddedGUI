# Stepper 示例

## 0. 为什么设计这个控件

`stepper` 用于表达离散步骤进度（当前步骤、总步骤、完成边界），适合向导流程、下单流程、激活流程等场景。  
相比通用指示器，它在业务语义上更明确，也更便于上层直接使用步骤 API。

## 0.1 为什么不用现有控件

- `page_indicator`：强调页面位置，不直接表达流程步骤语义。  
- `progress_bar`：偏连续进度，不适合离散步骤切换。  
- `tab_bar`：偏导航控件，作为轻量进度提示成本偏高。

设计取舍：绘制层复用 `page_indicator` 以保持资源开销可控，同时暴露 `stepper` 专用接口和场景边界。

## 1. 概述

本示例验证 `stepper` 在多步骤流程可视化中的显示与状态切换能力。

## 2. 视觉与布局规范

- Stepper 尺寸：`180x28`  
- 操作按钮尺寸：`120x36`  
- 布局：在 `220x150` 网格容器中垂直居中排布

## 3. 控件清单

| 变量名 | 类型 | 尺寸 | 初始状态 | 目的 |
|---|---|---|---|---|
| `stepper_view` | `egui_view_stepper_t` | 180x28 | `total=5,current=0` | 步骤进度展示 |
| `next_button` | `egui_view_button_t` | 120x36 | enabled | 推进步骤 |

## 4. 状态覆盖矩阵

| 状态 | 覆盖说明 |
|---|---|
| normal | 初始步骤 0 |
| updating | 点击按钮后递增当前步骤 |
| loop boundary | 最后一步后回绕到步骤 0 |

## 5. 录制动作

录制连续点击 `Next Step` 4 次，验证步骤过渡渲染。

## 6. 验收标准

- Runtime check 通过。  
- 交互转换检查通过且帧变化有效。  
- baseline 晋升后，回归检查通过。

## 7. 已知限制与下一步

- 当前示例通过外部按钮驱动步骤更新。  
- 如有需求可补充 stepper 直接触摸跳转能力。  
- 状态文字配色目前仅在完成态强调，后续可细化中间态表达。

## 8. 功能重叠与差异化边界

- 与 `page_indicator` 在绘制行为上有重叠。  
- 差异化目标是“流程步骤”语义、状态接口与业务可读性，而非页面分页导航。

## 9. 回归录制配置说明

- 推荐参数：`clock-scale=6`、`snapshot-stable-cycles=1`。  
- 必须覆盖回绕路径（`step 5 -> step 1`）以确保边界状态稳定。
