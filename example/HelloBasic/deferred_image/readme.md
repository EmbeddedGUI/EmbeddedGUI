# Deferred Image 控件演示

## 场景

这个示例演示一种特殊页面加载策略：

- 页面文字、卡片和布局先完成首屏渲染
- 文件图片区延后启动加载
- 加载失败时保持占位，不拖慢其它控件首帧

## 关键点

- 使用 `egui_view_deferred_image` 包装 `egui_image_file`
- 通过 `set_load_delay_ms()` 控制附着到窗口后的延迟启动
- 通过外部 `loader` 回调注入业务侧下载/缓存逻辑
- 示例里用一个假的异步 loader 模拟成功、慢成功和失败三种状态

## 录制动作

1. 先等待首屏稳定，并截取占位态
2. 再等待延迟加载完成，并截取最终态
3. 运行时自检成功卡片进入 `READY`，失败卡片进入 `FAILED`
