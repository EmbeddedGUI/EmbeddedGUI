# HTML to EmbeddedGUI Conversion Skill (v3 — XML Pipeline)

Convert Stitch HTML designs (Tailwind CSS + Material Symbols) or Figma Make JSX/TSX projects (React + Tailwind + Lucide) into EmbeddedGUI C code via XML layout files and the UI Designer code generator.

## Workflow Overview

```
1. extract-layout  -- Script parses HTML into structured JSON
2. scaffold        -- Script creates .egui project structure
3. export-icons    -- Script extracts icons to PNG (into .eguiproject + resource/src)
4. Write XML       -- AI writes main_page.xml layout (widget tree in XML)
5. generate-code   -- Script converts XML to C code (layout, headers, uicode)
6. gen-resource    -- Script generates C image/font arrays
7. Build & Verify  -- make + runtime check + screenshot comparison
```

**Key difference from v2**: AI writes XML layout instead of C code. The code generator handles all C code generation, ensuring correct API calls, struct definitions, and page lifecycle.

## Step 0: Scaffold & Resources

**Prerequisite**: Material Symbols Outlined font must be at `scripts/tools/MaterialSymbolsOutlined-Regular.ttf`.
Download from: https://github.com/google/material-design-icons/raw/master/variablefont/MaterialSymbolsOutlined%5BFILL%2CGRAD%2Copsz%2Cwght%5D.ttf

```bash
# Parse HTML structure (outputs JSON with sections, colors, icons)
python scripts/html2egui_helper.py extract-layout --input design.html

# Create .egui project structure
python scripts/html2egui_helper.py scaffold --app MyApp --width 320 --height 480

# Export icons (auto-detects colors/filled from HTML)
# With --app, icons go to .eguiproject/resources/images/ + resource/src/ automatically
python scripts/html2egui_helper.py export-icons --input design.html \
    --app MyApp --size 24 --auto-color
# For smaller status-bar icons:
python scripts/html2egui_helper.py export-icons --input design.html \
    --app MyApp --size 16 --color FFFFFF --suffix _16
# Or explicit output path:
python scripts/html2egui_helper.py export-icons --input design.html \
    --output example/MyApp/resource/src/ --size 24 --auto-color
```

## Step 1: Layout Analysis

Read the `extract-layout` JSON output. For each section, decide:
- **Widget type**: Group, Card, GridLayout, LinearLayout
- **Position**: x, y relative to parent
- **Size**: width, height in pixels
- **Children**: Labels, Images, sub-containers

Plan the widget tree before writing XML.

## Step 2: Write XML Layout

Edit `.eguiproject/layout/main_page.xml`. The root `<Group>` is the full-screen container.

### XML Widget Reference

#### Group (absolute positioning container)
Use for: root, status bars, headers, `justify-between` rows, badge pills.
Children positioned manually via x/y coordinates.

```xml
<Group id="root" x="0" y="0" width="320" height="480">
    <Label id="title" x="16" y="8" width="200" height="23" text="Hello" />
</Group>
```

#### Card (rounded corner container with background)
Use for: any `rounded-*` container with background color.

```xml
<Card id="device_card" x="16" y="200" width="288" height="100" corner_radius="12">
    <Background type="solid" color="EGUI_COLOR_HEX(0x1A2130)" alpha="EGUI_ALPHA_100" />
    <Label id="card_title" x="16" y="8" width="200" height="18" text="Device" />
</Card>
```

Card with border stroke:
```xml
<Card id="bordered_card" x="16" y="100" width="288" height="80" corner_radius="8">
    <Background type="round_rectangle" color="EGUI_COLOR_HEX(0x1A2130)" alpha="EGUI_ALPHA_100"
                radius="8" stroke_width="1" stroke_color="EGUI_COLOR_HEX(0x1E293B)" stroke_alpha="EGUI_ALPHA_100" />
    <Label id="inner" x="8" y="4" width="100" height="18" text="Inner" />
</Card>
```

#### LinearLayout (auto-stacking)
Use for: vertical/horizontal auto-layout sequences.

```xml
<!-- Vertical (default) -->
<LinearLayout id="vlist" x="0" y="0" width="288" height="300"
              orientation="vertical" align_type="EGUI_ALIGN_CENTER">
    <Label id="item1" x="0" y="0" width="100" height="23" text="Item 1" margin="4" />
    <Label id="item2" x="0" y="0" width="100" height="23" text="Item 2" margin="4" />
</LinearLayout>

<!-- Horizontal -->
<LinearLayout id="hlist" x="0" y="0" width="288" height="40"
              orientation="horizontal" align_type="EGUI_ALIGN_VCENTER">
    <Image id="icon" x="0" y="0" width="24" height="24" image_file="icon_wifi.png" />
    <Label id="name" x="0" y="0" width="100" height="23" text="WiFi" margin="4" />
</LinearLayout>
```

#### GridLayout (N-column grid)
Use for: `grid grid-cols-N` patterns.

```xml
<GridLayout id="grid" x="16" y="200" width="288" height="200"
            col_count="2" align_type="EGUI_ALIGN_CENTER">
    <Card id="cell1" x="0" y="0" width="136" height="90" corner_radius="12" margin="4">
        <Background type="solid" color="EGUI_COLOR_HEX(0x1A2130)" alpha="EGUI_ALPHA_100" />
        <Label id="cell1_text" x="16" y="8" width="100" height="18" text="Cell 1" />
    </Card>
    <!-- more cells... -->
</GridLayout>
```

#### Label
```xml
<Label id="title" x="16" y="4" width="180" height="23"
       text="Smart Home"
       font_builtin="&amp;egui_res_font_montserrat_18_4"
       color="EGUI_COLOR_WHITE" alpha="EGUI_ALPHA_100"
       align_type="EGUI_ALIGN_LEFT | EGUI_ALIGN_VCENTER" />
```

**Note**: `&` must be escaped as `&amp;` in XML. Example: `&amp;egui_res_font_montserrat_18_4`.

If `font_builtin` is omitted, the default font `EGUI_CONFIG_FONT_DEFAULT` is used.

#### Image (alpha-only icons — recommended for Material icons)
Icons store only alpha channel; color applied at render time. One resource in any color, ~5x smaller.

```xml
<Image id="icon_wifi" x="0" y="0" width="24" height="24"
       image_file="icon_wifi.png" image_format="alpha" image_alpha="4"
       image_color="EGUI_COLOR_WHITE" image_color_alpha="EGUI_ALPHA_100"
       image_resize="true" />
```

- `image_format="alpha"` + `image_alpha="4"`: alpha-only format
- `image_color` / `image_color_alpha`: render color (required for alpha-only icons)
- `image_resize="true"`: MUST set when view size != resource PNG size

#### Image (RGB565 — for full-color bitmaps)
```xml
<Image id="photo" x="0" y="0" width="100" height="100"
       image_file="photo.png" image_format="rgb565" image_alpha="4" />
```

#### CircularProgressBar (gauge/donut)
Calculate: `percent = (1 - stroke-dashoffset / stroke-dasharray) * 100`

```xml
<CircularProgressBar id="gauge" x="50" y="20" width="100" height="100"
                     value="74" stroke_width="6" />
```

#### ProgressBar
```xml
<ProgressBar id="progress" x="0" y="0" width="200" height="8" value="50" />
```

#### Switch
```xml
<Switch id="toggle" x="0" y="0" width="48" height="24" is_checked="true" />
```

### Background Element

Backgrounds are optional child elements of Card, Button, and other container widgets:

```xml
<!-- Solid color -->
<Background type="solid" color="EGUI_COLOR_HEX(0x101622)" alpha="EGUI_ALPHA_100" />

<!-- Rounded rectangle -->
<Background type="round_rectangle" color="EGUI_COLOR_HEX(0x1A2130)" alpha="EGUI_ALPHA_100"
            radius="12" />

<!-- Rounded rectangle with border -->
<Background type="round_rectangle" color="EGUI_COLOR_HEX(0x1A2130)" alpha="EGUI_ALPHA_100"
            radius="12" stroke_width="1" stroke_color="EGUI_COLOR_HEX(0x1E293B)"
            stroke_alpha="EGUI_ALPHA_100" />

<!-- With pressed state (for buttons) -->
<Background type="round_rectangle" color="EGUI_COLOR_BLUE" alpha="EGUI_ALPHA_100" radius="8"
            has_pressed="true" pressed_color="EGUI_COLOR_NAVY" pressed_alpha="EGUI_ALPHA_100" />
```

### Complete XML Example

```xml
<?xml version="1.0" encoding="utf-8"?>
<Page>
    <Group id="root" x="0" y="0" width="320" height="480">
        <!-- Dark background applied to root -->
        <Background type="solid" color="EGUI_COLOR_HEX(0x0F172A)" alpha="EGUI_ALPHA_100" />

        <!-- Status bar -->
        <Group id="status_bar" x="0" y="0" width="320" height="32">
            <Label id="time_label" x="16" y="4" width="80" height="23"
                   text="12:30" font_builtin="&amp;egui_res_font_montserrat_14_4"
                   color="EGUI_COLOR_WHITE" alpha="EGUI_ALPHA_100"
                   align_type="EGUI_ALIGN_LEFT | EGUI_ALIGN_VCENTER" />
            <Image id="wifi_icon" x="280" y="4" width="24" height="24"
                   image_file="icon_wifi.png" image_format="alpha" image_alpha="4"
                   image_color="EGUI_COLOR_WHITE" image_color_alpha="EGUI_ALPHA_100" />
        </Group>

        <!-- Title -->
        <Label id="title" x="16" y="40" width="200" height="31"
               text="Smart Home" font_builtin="&amp;egui_res_font_montserrat_24_4"
               color="EGUI_COLOR_WHITE" alpha="EGUI_ALPHA_100" />

        <!-- Device grid -->
        <GridLayout id="devices" x="16" y="80" width="288" height="200"
                    col_count="2" align_type="EGUI_ALIGN_CENTER">
            <Card id="light_card" x="0" y="0" width="136" height="90"
                  corner_radius="12" margin="4">
                <Background type="solid" color="EGUI_COLOR_HEX(0x1A2130)"
                            alpha="EGUI_ALPHA_100" />
                <Image id="light_icon" x="16" y="12" width="24" height="24"
                       image_file="icon_lightbulb.png" image_format="alpha" image_alpha="4"
                       image_color="EGUI_COLOR_HEX(0xFBBF24)" image_color_alpha="EGUI_ALPHA_100" />
                <Label id="light_name" x="16" y="44" width="100" height="18"
                       text="Lights" font_builtin="&amp;egui_res_font_montserrat_14_4"
                       color="EGUI_COLOR_WHITE" alpha="EGUI_ALPHA_100" />
                <Label id="light_status" x="16" y="64" width="100" height="16"
                       text="On" font_builtin="&amp;egui_res_font_montserrat_12_4"
                       color="EGUI_COLOR_HEX(0x94A3B8)" alpha="EGUI_ALPHA_100" />
            </Card>
        </GridLayout>
    </Group>
</Page>
```

### XML Rules (NEVER violate)

| # | Rule |
|---|------|
| 1 | Root element is `<Page>`, direct child is a `<Group id="root">` spanning the screen |
| 2 | All widgets MUST have `id`, `x`, `y`, `width`, `height` attributes |
| 3 | `id` values must be unique within the page and valid C identifiers (lowercase, underscores) |
| 4 | Font references use `&amp;` (XML escape): `font_builtin="&amp;egui_res_font_montserrat_18_4"` |
| 5 | Alpha-only images (`image_format="alpha"`) MUST set `image_color` and `image_color_alpha` |
| 6 | When image view size != resource PNG size, set `image_resize="true"` |
| 7 | Label height MUST be >= font line height (see font table below) |
| 8 | Card children are auto-positioned by `layout_childs`. Use margin for spacing |
| 9 | GridLayout/LinearLayout children are auto-positioned. Set `margin` on children for gaps |
| 10 | Group children use explicit x/y positioning (no auto-layout) |
| 11 | `<Background>` is an optional child element of containers — not an attribute |
| 12 | Colors use EGUI constants: `EGUI_COLOR_WHITE`, `EGUI_COLOR_HEX(0xRRGGBB)` |

## Step 3: Generate Code

```bash
# Generate C code from XML layout
python scripts/html2egui_helper.py generate-code --app MyApp

# Generate C resources (IMPORTANT: delete old img/*.c first if re-exporting icons)
rm -f example/MyApp/resource/img/*.c example/MyApp/resource/app_egui_resource_generate.*
python scripts/html2egui_helper.py gen-resource --app MyApp
```

The `generate-code` command:
1. Loads the `.egui` project file
2. Scans XML widgets to auto-generate `app_resource_config.json` (image format/size info)
3. Generates C files:
   - `main_page.h` — Page struct with widget members
   - `main_page_layout.c` — Layout init function (100% auto-generated, always overwritten)
   - `main_page.c` — User implementation skeleton (created once, NEVER overwritten)
   - `uicode.h/c` — Page management (page switching, startup)
   - `app_egui_config.h` — Screen config (only if not present)

## Step 4: Build & Verify

```bash
# One-command build + runtime check
python scripts/html2egui_helper.py verify --app MyApp

# Or separately:
make all APP=MyApp PORT=pc BITS=64
python scripts/code_runtime_check.py --app MyApp --bits64 --keep-screenshots
```

Read the screenshot from `runtime_check_output/MyApp/` and compare with the design PNG.

### Common Fixes

| Symptom | Fix |
|---------|-----|
| Elements overlap | Increase container size or reduce child sizes |
| Wrong spacing | Adjust `margin` attribute on children |
| Wrong color | Check `EGUI_COLOR_HEX(0xRRGGBB)` — RGB order |
| Text too big/small | Change `font_builtin` size |
| Text cut off right | Increase label width |
| Text cut off bottom | Label height < font line height. See font table below |
| Icon cropped | Set `image_resize="true"` in XML |
| Character shows as box | Not in font charset. Check `supported_text.txt` |
| Element invisible | Check: alpha, size > 0, widget is child of root tree |
| Card bg not showing | Add `<Background>` child element |
| Icon blank | Check `image_file` matches PNG filename exactly |
| justify-between | Use Group + manual x/y positions, NOT LinearLayout |

### Iteration: Fix XML, Not C

When fixing rendering issues, edit the XML file and re-run:
```bash
python scripts/html2egui_helper.py generate-code --app MyApp
make all APP=MyApp PORT=pc BITS=64
python scripts/code_runtime_check.py --app MyApp --bits64 --keep-screenshots
```

The layout C code is fully regenerated each time. Never edit `*_layout.c` manually.

## Step 5: Rendering Verification & Optimization

After initial build passes, compare the screenshot with the design and fix clipping/rendering issues. EGUI clips all rendering to the parent view bounds.

### 5.1 Check for Image Clipping

**Symptom**: Icon appears cropped or partially missing.

**Fix**: Set `image_resize="true"` in XML when view size != resource PNG size.

### 5.2 Check for Text Clipping

**Font line height table** (MUST use these as minimum label heights):

| Font | Line Height (px) |
|------|-----------------|
| `montserrat_8_4` | **11** |
| `montserrat_10_4` | **13** |
| `montserrat_12_4` | **16** |
| `montserrat_14_4` | **18** |
| `montserrat_16_4` | **21** |
| `montserrat_18_4` | **23** |
| `montserrat_20_4` | **26** |
| `montserrat_24_4` | **31** |
| `montserrat_30_4` | **38** |
| `montserrat_36_4` | **45** |
| `montserrat_48_4` | **60** |

### 5.3 Check for Container Overflow

Ensure parent container size accommodates all children:
```
parent.height >= max(child.y + child.height) for all children
parent.width  >= max(child.x + child.width)  for all children
```

### 5.4 Iteration Workflow

```
1. Build + runtime check → take screenshot
2. Compare screenshot with design:
   - All icons visible? (set image_resize="true")
   - All text readable? (check label height >= font line height)
   - Containers show all children? (increase container size)
   - Positions correct? (adjust x/y/width/height)
3. Edit XML, re-run generate-code + build
4. Repeat until screenshot matches design
```

## Figma Make Workflow

Figma Make generates React (TSX) + Tailwind CSS projects with Lucide icons.
Use the `figmamake-extract` subcommand to parse these projects.

### Figma Make vs Stitch HTML Differences

| Aspect | Stitch HTML | Figma Make JSX |
|--------|-------------|----------------|
| Attributes | `class` | `className` |
| Icons | Material Symbols `<span>` | Lucide React `<Battery />` |
| Dynamic content | Static text | `{variable}`, `.map()`, conditionals |
| Structure | Single HTML file | Multi-file components + routes |
| Comments | `<!-- -->` | `{/* */}` |
| Opacity | None | `bg-gray-800/50`, `border-cyan-500/30` |

### Figma Make Workflow Steps

```
1. figmamake-extract  -- Parse JSX/TSX project to structured JSON
2. scaffold           -- Create .egui project structure
3. export-icons       -- Export Material Symbols PNGs (using mapped icon names)
4. Write XML          -- AI writes EGUI XML layout from JSON
5. generate-code      -- XML to C code
6. gen-resource       -- Generate C resource arrays
7. verify             -- Build + runtime check
```

### Step 0: Discover Pages

```bash
# List all pages in the project
python scripts/html2egui_helper.py figmamake-extract \
    --input figma_make_battery/ --list-pages

# Extract a specific page
python scripts/html2egui_helper.py figmamake-extract \
    --input figma_make_battery/ --page Home

# Extract all pages
python scripts/html2egui_helper.py figmamake-extract \
    --input figma_make_battery/
```

### JSON Output Format

The `figmamake-extract` output is compatible with `extract-layout`, with additions:

```json
{
  "source": "figma-make",

  "root_bg_color": "0A0E1A",
  "colors": {"cyan-400": "22D3EE", "gray-400": "9CA3AF"},
  "icons": [
    {"lucide": "Battery", "material": "battery_full", "mapped": true,
     "color": "cyan-400", "size_px": 20}
  ],
  "pages": [{
    "name": "Home",
    "route": "/",
    "sections": [...]
  }],
  "text_chars": "..."
}
```

### Icon Handling

Lucide React icons are automatically mapped to Material Symbols equivalents:

| Lucide | Material Symbols |
|--------|-----------------|
| `Battery` | `battery_full` |
| `Zap` | `bolt` |
| `ThermometerSun` | `thermostat` |
| `Settings` | `settings` |
| `ArrowLeft` | `arrow_back` |
| `Wifi` | `wifi` |
| `AlertTriangle` | `warning` |

Use the mapped Material Symbols names with `export-icons`:
```bash
# After figmamake-extract, create the icon list from JSON
python scripts/html2egui_helper.py export-icons \
    --input design.html --app MyApp --size 24 --auto-color
```

For Figma Make projects without HTML, manually export icons using the mapped names
from the JSON `icons` array.

### Dynamic Content Handling

- `{variable}` expressions become placeholder text (e.g., `batteryLevel`)
- `.map()` iterations extract the template element (shown once)
- Conditional rendering (`{cond && <Elem/>}`) keeps the element
- Template literal classNames resolve to the first branch

The AI should interpret these placeholders when writing XML and use
appropriate static values or format strings.

### Known Limitations

1. **Recharts components** (`LineChart`, `AreaChart`): Marked as `data-component` containers. Use Group + Label approximation in XML.
2. **CSS variable colors** (`oklch()` in `theme.css`): Not parsed. Components use inline Tailwind classes directly.
3. **Gradient backgrounds** (`bg-gradient-to-br`): Approximated as solid color (first color).
4. **hover/transition states**: Stripped (EGUI uses Background pressed state).

## Multi-Page Projects

### Project Structure

Multi-page projects have multiple XML files in `.eguiproject/layout/`:

```
.eguiproject/layout/
├── main_page.xml         ← Startup page (first in .egui file)
├── battery_detail.xml    ← Detail page
├── temperature.xml       ← Sub-page
└── settings.xml          ← Settings page
```

Each `<PageRef>` in the `.egui` project file registers a page:
```xml
<Project app_name="MyApp" screen_width="320" screen_height="240"
         page_mode="easy_page" startup="main_page" egui_root="../..">
    <Pages>
        <PageRef file="layout/main_page.xml" />
        <PageRef file="layout/battery_detail.xml" />
        <PageRef file="layout/temperature.xml" />
        <PageRef file="layout/settings.xml" />
    </Pages>
    <Resources catalog="resources.xml" />
</Project>
```

Use `scaffold --pages` to create multi-page projects:
```bash
python scripts/html2egui_helper.py scaffold --app MyApp --width 320 --height 240 \
    --pages main_page,battery_detail,temperature,settings
```

### Page Enum (Auto-Generated)

The code generator creates a page index enum in `uicode.h`:
```c
enum {
    PAGE_MAIN_PAGE = 0,
    PAGE_BATTERY_DETAIL = 1,
    PAGE_TEMPERATURE = 2,
    PAGE_SETTINGS = 3,
    PAGE_COUNT = 4,
};
```

### Navigation via onClick

Any widget can trigger page navigation using the `onClick` XML attribute.

**Forward navigation (e.g., Home → Detail):**
```xml
<Card id="battery_card" x="8" y="28" width="304" height="76" corner_radius="8"
      onClick="on_battery_card_click">
    <!-- Card content -->
</Card>
```

The callback is defined in the user code file (`main_page.c`):
```c
static void on_battery_card_click(egui_view_t *self)
{
    uicode_switch_page(PAGE_BATTERY_DETAIL);
}
```

**Back navigation (e.g., Detail → Home):**
```xml
<Image id="btn_back" x="0" y="0" width="16" height="16"
       image_file="icon_arrow_back.png" image_format="alpha" image_alpha="4"
       image_color="EGUI_COLOR_WHITE" image_color_alpha="EGUI_ALPHA_100"
       image_resize="true" onClick="on_back_click" />
```

```c
static void on_back_click(egui_view_t *self)
{
    uicode_switch_page(PAGE_MAIN_PAGE);
}
```

### Navigation from figmamake-extract

The `figmamake-extract` command outputs navigation info from `navigate()` calls:
```json
{
  "navigation": [
    {"from": "/", "to": "/battery", "trigger": "section_1"},
    {"from": "/battery", "to": "/", "trigger": "container"}
  ]
}
```

Map routes to page names based on the `pages[].route` field.

### Multi-Page Workflow

```
1. figmamake-extract --input project/          # Extract all pages (includes navigation)
2. scaffold --app MyApp --pages p1,p2,p3       # Create multi-page project
3. export-icons                                 # Export icons for all pages
4. Write XML for each page                      # Include onClick attributes
5. generate-code                                # Auto-generates page enum + switch logic
6. Edit *.c files — add navigation callbacks    # onClick callbacks call uicode_switch_page()
7. gen-resource → verify                        # Build and verify
```

### Generated File Structure (per page)

| File | Type | Overwritten? | Purpose |
|------|------|-------------|---------|
| `{page}.h` | Generated | Yes (user code preserved) | Page struct with widget members |
| `{page}_layout.c` | Generated | Always | Layout init code (100% auto-generated) |
| `{page}.c` | User-owned | Created once, never overwritten | User callbacks and custom logic |

### Rules for Multi-Page XML

| # | Rule |
|---|------|
| 1 | Each page XML has its own `<Group id="root">` spanning the full screen |
| 2 | Each page should have its own dark background `<Background>` |
| 3 | Widget `id` values must be unique **within each page** (can repeat across pages) |
| 4 | `onClick` attribute value is a C function name (valid C identifier) |
| 5 | Navigation callbacks go in the user code file (`{page}.c`), not layout |
| 6 | The `uicode_switch_page()` function is auto-generated — never write it manually |
